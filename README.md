# Industrial Technical RAG Assistant  
**Локальный ИИ-ассистент по технической документации и ремонту оборудования**  
(на базе LlamaIndex + Ollama + Chroma)

![Industrial RAG Demo](https://via.placeholder.com/800x400?text=Industrial+RAG+Assistant)  
*(Здесь можно вставить скриншот интерфейса или схему пайплайна)*

## О проекте

Полностью **локальный** (on-premise / air-gapped) RAG-чатбот, который отвечает на вопросы рабочих, инженеров и наладчиков **только по внутренним документам предприятия**:

- инструкции по эксплуатации станков и агрегатов  
- журналы ремонтов и отчёты о простоях  
- регламенты ТОиР, ТБ, фото дефектов с описаниями  
- технологические карты и схемы  

**Цели проекта**:
- Сократить время поиска информации в документации на 30–70 %  
- Уменьшить простои оборудования за счёт быстрого доступа к релевантным знаниям  
- Полная конфиденциальность — никакие данные не уходят наружу  

Работает на ноутбуке/edge-сервере в цеху (тестировалось на RTX 3050 Mobile 4 ГБ + 16 ГБ RAM).

## Технологический стек

- **LLM** — Ollama (qwen2.5:7b-instruct-q4_K_M или q5_K_M)  
- **Embedding** — BAAI/bge-m3 (мультиязычная, отличная поддержка русского)  
- **Векторная БД** — Chroma (локальная, persistent)  
- **RAG-фреймворк** — LlamaIndex  
- **Интерфейс** — консольный (пока), легко расширить до Gradio / Chainlit / Open WebUI  
- **Язык** — Python 3.10–3.12  

## Требования

- ОС: Linux / Windows / macOS  
- GPU: NVIDIA с ≥ 4 ГБ VRAM (RTX 3050 Mobile и выше)  
- RAM: ≥ 12 ГБ (рекомендуется 16 ГБ)  
- Ollama установлен и запущен[](https://ollama.com)  
- Python 3.10+ с venv  

## Быстрый старт

1. Клонируйте репозиторий
   ```bash
   git clone https://github.com/ainureg/industrial-rag.git
   cd industrial-rag
   ```
2. Создайте виртуальное окружение и установите зависимости 
```Bash
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
```
3. Скачайте модель (если ещё не скачана)
```Bash
ollama pull qwen2.5:7b-instruct-q4_K_M
```
4. Положите документы в папку data/
 - PDF, DOCX, TXT, MD
 - Рекомендуется: инструкции, отчёты о ремонтах, фото с подписями

5. Постройте индекс (один раз или при добавлении документов)
```Bash
python 2_build_index.py
```
6. Запустите чатбот
```Bash
python 3_rag_pipeline.py
```
Пример диалога:text
```
Вопрос: Как устранить течь в гидроцилиндре станка АБВ-123?
Ответ: Согласно инструкции И-456/2024, необходимо...
Источники: data/instrukciya_abv123.pdf ...
```
Структура проекта
```text
industrial-rag/
├── data/                  # ← кладите сюда все документы (PDF, DOCX, TXT, изображения)
├── chroma_db/             # векторная БД Chroma (создаётся автоматически)
├── 1_prepare_data.py      # (опционально) предобработка и чанкинг
├── 2_build_index.py       # создание/обновление индекса
├── 3_rag_pipeline.py      # основной файл — запуск чатбота
├── app.py                 # (будущий) веб-интерфейс на Gradio / Chainlit
├── requirements.txt       # зависимости
├── README.md              # этот файл
└── .gitignore
```
Возможные расширения

- Голосовой ввод — Faster-Whisper / Whisper.cpp
- Анализ фото дефектов — Qwen2-VL-7B-Instruct (Q4_K_M)
- Multi-agent — LangGraph (Retriever → Reporter → Critic)
- Веб-интерфейс — Gradio, Chainlit, Open WebUI
- Автообновление индекса — watchdog + периодическая переиндексация
- Оценка качества — RAGAS / ручная оценка по 30–50 вопросам
